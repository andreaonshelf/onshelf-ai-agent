2. SVG Rendering API

Location: /src/api/planogram_renderer.py

Features:

  • Server-side SVG generation from extraction data
  • Three abstraction levels: brand_view, product_view, sku_view
  • Color-coded confidence visualization
  • Scalable vector graphics output
GET /api/planogram/{image_id}/render?abstraction_level=product_view&format=svg
# SVG Generation Example def generate_svg_from_real_data(extraction_data, abstraction_level): products = extraction_data.get("products", []) # Color based on confidence if confidence >= 0.9: color = "#28a745" # Green elif confidence >= 0.7: color = "#ffc107" # Yellow else: color = "#dc3545" # Red # Generate SVG elements for each product svg_content = f''' <rect x="{x}" y="{y}" width="80" height="70" fill="{color}" stroke="#333"/> <text x="{x+40}" y="{y+35}" text-anchor="middle"> {product_name} </text> '''

3. PNG Rendering API

Location: /src/api/planogram_renderer.py

Features:

  • Server-side PNG generation using PIL/Pillow
  • Base64 encoded output for web display
  • Suitable for email reports or static exports
  • Fixed dimensions with product list layout
GET /api/planogram/{image_id}/render?abstraction_level=product_view&format=png
# PNG Generation Example def generate_png_from_real_data(extraction_data, abstraction_level): image = Image.new('RGB', (800, 600), 'white') draw = ImageDraw.Draw(image) # Draw products with confidence colors for product in products: if confidence >= 0.9: color = (40, 167, 69) # Green elif confidence >= 0.7: color = (255, 193, 7) # Yellow else: color = (220, 53, 69) # Red draw.rectangle([x, y, x+width, y+height], fill=color, outline=(0, 0, 0))

5. Complete API Endpoints

Planogram Rendering:

GET /api/planogram/{image_id}/render

Parameters:

  • abstraction_level: brand_view, product_view, sku_view
  • format: svg, png, json

Editable Planogram Data:

GET /api/planogram/{image_id}/editable

Returns planogram data merged with human corrections for interactive editing

Save Corrections:

POST /api/planogram/{image_id}/corrections

Saves human corrections for planogram edits

Demo Access:

GET /api/planogram/demo/editable

Returns the full 18-product demo planogram for testing

Key Differences Between Approaches

Approach Best For Key Features
React Component Interactive dashboard views Real-time updates, toggle overlays, hover effects
SVG API Web display, scalable graphics Clean vectors, customizable abstraction
PNG API Reports, emails, static exports Fixed format, base64 encoding
JSON API Data exchange, custom rendering Raw data access, flexible integration
