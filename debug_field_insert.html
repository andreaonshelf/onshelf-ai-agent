Debug Field Insert Feature

Open your main dashboard in a new tab, then open the browser console and run these commands to debug:

// 1. Check if insert buttons are in the DOM
console.log('Insert buttons found:', document.querySelectorAll('.insert-field-button').length);

// 2. Check opacity of insert wrappers
document.querySelectorAll('.insert-field-wrapper').forEach((el, i) => {
    console.log(`Insert wrapper ${i} opacity:`, el.style.opacity);
});

// 3. Force show all insert buttons
document.querySelectorAll('.insert-field-wrapper').forEach(el => {
    el.style.opacity = '1';
    el.style.border = '2px solid red';
});

// 4. Check if hover events are working
document.querySelectorAll('.insert-field-wrapper').forEach((el, i) => {
    el.addEventListener('mouseenter', () => console.log(`Hovering insert ${i}`));
});

// 5. Check z-index issues
document.querySelectorAll('.insert-field-wrapper').forEach(el => {
    el.style.zIndex = '9999';
});

Common Issues:

  • Buttons not visible: Run command #3 to force show them
  • Hover not working: Check console for hover logs (command #4)
  • Z-index issues: Run command #5 to bring buttons to front

Quick Fix - Add this CSS:

/* Add to your dashboard styles */
.field-editor-container .insert-field-wrapper {
    opacity: 0.3 !important;
    display: flex !important;
}

.field-editor-container .insert-field-wrapper:hover {
    opacity: 1 !important;
}

/* Always show first insert button */
.field-editor-container > .insert-field-wrapper:first-child {
    opacity: 0.5 !important;
}
