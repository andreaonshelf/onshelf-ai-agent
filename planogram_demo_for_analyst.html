OnShelf Planogram Visualization Example

Reference for AI Prompt Engineering

Important: This is a simplified 2D representation for CPG marketers to understand shelf execution. NOT a pixel-perfect replica of the actual shelf photo. Products are shown as boxes with key information only.

What Our AI Extracts and How It's Visualized

Key Extraction Points:

  • Shelf Structure: Number of shelves (counted bottom to top)
  • Product Position: Which shelf + left-to-right position
  • Facings: How many units side-by-side (3 facings = 3 cells in grid)
  • Gaps: Empty spaces between products (shown as dashed boxes)
  • Product Info: Brand, name, price, size
  • Confidence: How certain the AI is (shown by color)

Critical Points for Prompt Writing

Remember When Writing Prompts:

  • Grid Awareness: Each facing = 1 cell, gaps = empty cells
  • Position Criticality: Wrong positions create wrong visualizations
  • Shelf-by-Shelf: Extract one shelf at a time for accuracy
  • Business Context: This helps CPG marketers optimize execution
  • Not Pixel-Perfect: We need abstracted data, not exact replication

Example Prompt Structure

You are extracting products that will be rendered in a visual planogram grid.

CRITICAL: Your extraction will be visualized as follows:
- Each FACING gets its own grid cell (3 facings = 3 separate cells)
- Empty positions will be transparent gaps
- Position numbers determine left-to-right order

For Shelf {N}, extract:
1. Products from left to right
2. Brand and product name
3. Facing count (how many side-by-side)
4. Price if visible
5. Note any gaps between products

Remember: This creates a simplified 2D grid for marketers, not a photo.
